Beam Centers

I use the mirror parameters derived from the Moon Analysis described in JAC2022c and fit for a source location over all DK angles using a method similar to how we fit for mirror parameters described in § 2 of except we take the mirror parameters in the pointing model as known and instead let the source azimuth and elevation be free parameters:

With beam_map_pointing_model.m as our model we computer the instrument-fixed coordinates in x/y:

\begin{equation} [x,\,y,\,\phi] = \text{beam_map_pointing_model}(Az_{src},\,El_{src} | A_{0},\,E_{0},\,K_{0},\,Mount,\,Mirror) \end{equation} where $A_0$, $E_0$, and $D_0$ are the as-fit beam centers in raw mount coordinates; $Mount$ comprises the relevant mount information derived from both optical starpointing and Solidworks drawings; $Mirror$ comprises the tilt, roll and height of the mirror.

Then using CMB-derived beam centers as the 'observed' quantity, the chi-square to minimize is just:

\begin{equation} \chi^2=\sum \frac{(x_{CMB,i}-x_{0,i})^2+(y_{CMB,i}-y_{0,i})^2}{\sigma_i^2} \end{equation}

From JAC2022c, we have evidence that the mirror is physically moving about from measurement-to-measurement, so after fitting an overall source position I keep it fixed and fit for the mirror parameters per-observation (and per-scan where applicable).

  • Y-Axis - Dependent variable. FPU Angle/Scale overall uses a fixed parameter for the mirror location whereas FPU Angle/Scale per obs uses the per-obs (or per-scan) fits for the mirror when calculating x/y coordinates to derive the angle/scaling.
  • X-Axis - Independent variable.
  • Target - Switch between Moon and RPS Type 5 data. $x_m$, $y_m$ and $\theta_m$ are the beam centeroids with the DK-angle derotated to get us into a quasi-mirror-fixed coordinate system.
  • Resolution - A data point represents either an entire observation or a single scan (or rasterset) within that observation.
  • Early Moon Data Cut - Use mirror parameters from with JAC2022c Table 2.2 with the first bit of data cut.
