Issues with Az/El tilts

The RPS residuals look pretty strange compared to the mirror best-fit residuals.

Looking at the RPS scans, we can see that both Azimuth and Deck appear to diverge at the elevation approaches 90°. This results in the scan apparently bunching up in boresight-centered coordinates.

This is because the solutions for the Azimuth and Elevation tilts become undefined when the telescope is pointing around Zenith. I didn't notice this in the moon data because the moon is higher on the horizon than the RPS.

The keck_beam_map_pointing code has functionality to apply Az/El tilts. Without a mirror, it still has the same problem when the telescope is pointed at zenith. But this problem goes away when using a mirror since the apparent az/el/pa are calculated after the mirror rotation when the apparent pointing is far away from Zenith.

So I've elected instead to apply the tilts through keck_beam_map_pointing for moon analysis. The best-fit roll parameter changes by O(0.1°)!

Applying the change to source derivation analysis, we see the beam-center residuals drastically reduced and appear to be more akin to the residuals we see in the mirror.
